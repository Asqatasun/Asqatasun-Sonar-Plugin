Il est parfois compliqué pour les utilisateurs de technologies d'assistance de déterminer la portée d'un champ de formulaire. Une étiquette seule, peut parfois ne pas suffire.

La légende des <fieldset> va être utilisée pour être enoncé par les lecteurs d'écran chaque fois qu'un champ va prendre le focus. La légende doit être la plus concise possible, et doit éviter de reprendre des termes déjà présents dans les étiquettes des champs.

Exemple de code non conforme

<fieldset>                                 <!-- Non-Conforme -->
  Nom: <input type="text"><br />
  Prénom: <input type="text"><br />
</fieldset>

Exemple de code conforme

<fieldset>                                 <!-- Conforme -->
  <legend>Identité</legend>
  Nom: <input type="text"><br />
  Prénom: <input type="text"><br />
</fieldset>
